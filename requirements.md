Сделай полноценный **MVP веб-чата с интерфейсом в стиле Telegram**, но с упором на **максимальную простоту стека, легкость запуска на слабом Ubuntu-сервере и поддержку около 10 одновременных пользователей**.

## Главная цель

Нужен **готовый к запуску self-hosted чат** для небольшой группы пользователей. Это должен быть не “концепт”, а **практичный MVP**, который можно развернуть на Ubuntu и использовать сразу.

## Ключевые ограничения

* сервер слабый, без тяжелой инфраструктуры
* пользователей немного: **до 10 человек**
* нужен **минимально сложный стек**
* вся история сообщений хранится **на сервере**
* история **не удаляется автоматически**
* историю можно **просматривать, листать, догружать**
* обязательна **мобильная адаптация**
* интерфейс должен быть **темный, визуально близкий к Telegram**
* решение должно быть **максимально простым в поддержке**

## Технологические требования

Выбери **самый легкий и практичный стек** для такой задачи. Предпочтительно:

* **Backend:** Node.js + Express
* **Realtime:** WebSocket (`ws`)
* **БД:** SQLite
* **Frontend:** легкий веб-интерфейс без тяжелых зависимостей
* **Deploy:** Ubuntu + Nginx + PM2 или systemd

Если предлагаешь другой стек, он должен быть **объективно проще и легче**, чем этот.

## Что нужно реализовать

Сделай MVP с максимально возможным количеством **реально нужных** функций, похожих на Telegram, но без избыточной сложности.

### Обязательно:

### 1. Регистрация и авторизация

* вход по логину/паролю
* хранение паролей только в хешированном виде
* защита сессий

### 2. Список чатов / комнат

* боковая панель со списком чатов
* отображение последнего сообщения
* отображение времени последнего сообщения

### 3. Чаты

* минимум один общий чат обязателен
* поддержка нескольких комнат, если это не усложняет проект
* личные чаты, общие и закрытые чаты.

### 4. Реалтайм сообщения

* WebSocket
* мгновенная доставка сообщений

### 5. История сообщений

* хранится в SQLite
* не удаляется
* догрузка старых сообщений при скролле вверх
* пагинация

### 6. Файлы и вложения

Обязательно реализовать загрузку и отображение вложений через локальное хранение на сервере, без S3 и без внешних сервисов.

Поддерживаемые типы:

* изображения: jpg, jpeg, png, webp, gif
* документы: pdf, txt, doc, docx, xls, xlsx, zip
* аудио: mp3, wav, ogg, m4a
* видео: mp4, webm, mov

Требования:

* загрузка файлов через сервер
* хранение локально на сервере
* ограничение размера файла, например 25MB
* серверная валидация типа и размера
* безопасные имена файлов
* в БД хранить метаданные файла

### 7. Изображения

Для изображений обязательно:

* inline preview прямо в сообщении
* клик для открытия в полном размере
* корректное отображение на мобильных устройствах
* сохранение размеров и метаданных при необходимости

### 8. Аудио

Для аудио обязательно:

* встроенное проигрывание прямо в сообщении через HTML5 audio player
* отображение имени файла
* возможность скачать файл
* корректная работа в мобильном браузере

### 9. Видео

Для видео обязательно:

* встроенное проигрывание прямо в сообщении через HTML5 video player
* poster/preview если возможно без усложнения
* адаптивное отображение в сообщении
* возможность открыть или скачать файл
* корректная работа на мобильных устройствах

### 10. Файлы

Для обычных файлов:

* отображение имени файла
* размер файла
* кнопка скачивания

### 11. Ссылки с предпросмотром (link preview)

При отправке сообщения со ссылкой:

* backend должен извлекать мета-данные страницы
* вытягивать по возможности:

  * title
  * description
  * preview image
  * hostname
* сохранять preview в БД
* показывать карточку ссылки внутри сообщения

Важно:

* использовать простой fetch + парсинг meta тегов
* не добавлять тяжелые внешние сервисы
* если preview получить не удалось, ссылка должна просто отображаться как обычная ссылка

### 12. Эмодзи

Обязательно:

* поддержка эмодзи из клавиатуры
* легкий emoji picker без тяжелых зависимостей
* корректное отображение эмодзи в сообщениях и поле ввода

### 13. UI (темный Telegram-like)

* темная тема по умолчанию
* стиль максимально близкий к Telegram
* боковая панель со списком чатов
* окно чата
* пузырьки сообщений
* разные стили своих и чужих сообщений
* аккуратные карточки ссылок
* аккуратное отображение вложений
* фиксированное поле ввода снизу
* современный чистый внешний вид
* без перегруженного UI

### 14. Мобильная адаптация

Обязательно:

* полноценная работа на телефоне
* адаптивная верстка
* корректное открытие/закрытие боковой панели
* удобная работа со списком чатов
* корректное отображение изображений, видео и аудио
* удобная кнопка прикрепления файла
* удобное поле ввода на маленьких экранах
* отсутствие поломок верстки

### 15. Онлайн-статус

* кто в сети
* базовое presence пользователей

### 16. Админ-функции

* удаление сообщений
* блокировка пользователя
* просмотр пользователей

### 17. Безопасность

* bcrypt/argon2 для паролей
* валидация входных данных
* защита от XSS
* ограничение длины сообщений
* rate limit
* проверка mime/type файлов
* защита от загрузки опасных файлов
* раздача файлов через безопасный статический маршрут
